# Bowling Game Kata — Plan

## 요구사항 요약 (CLAUDE.md 기준)

- `Game` 클래스는 `roll(pins: int) -> None` 과 `score() -> int` 두 메서드만 노출한다.
- 개별 roll의 유효성(핀 개수 범위, 프레임 합계 등)은 검증하지 않는다.
- 10프레임, 프레임당 기본 2회 롤. 단, 스트라이크는 1회, 10프레임의 스페어/스트라이크는 필 볼(fill ball)로 최대 3회.
- 스코어링 규칙:
  - 스페어(한 프레임 두 롤 합 10): 10 + 다음 1롤
  - 스트라이크(첫 롤이 10): 10 + 다음 2롤
  - 10프레임은 필 볼을 포함해 위 보너스를 그대로 적용하되, 11프레임은 없음.

## 구현 방향 (강제하지 않음, GREEN 단계에서 결정)

- `roll()`은 단순히 굴린 핀 수를 리스트에 누적한다.
- `score()`는 그 리스트를 프레임 경계를 따라가며 순회해 계산한다 (스트라이크/스페어 보너스가 다음 프레임, 혹은 필 볼을 넘겨봐야 하므로).

## 테스트 계획 (RED 순서, 단순 → 복잡)

| # | 테스트 이름 | 검증 내용 | 이전 테스트로 커버되지 않는 이유 |
|---|---|---|---|
| 1 | `test_all_gutters_scores_zero` | 모든 프레임이 거터(0점)일 때 `score() == 0` | 가장 단순한 케이스, `Game` 클래스 존재 여부부터 확인 |
| 2 | `test_all_ones_scores_twenty` | 20회 모두 1핀 → `score() == 20` | 보너스 없는 "단순 합산" 로직 검증 (거터만으로는 합산 자체를 증명 못함) |
| 3 | `test_one_spare_scores_bonus` | 한 프레임에서 스페어(5+5) 후 다음 롤 3, 나머지 거터 → `score() == 10 + 3 + 3 = 16` | 스페어 보너스(다음 1롤 가산) 로직은 아직 검증되지 않음 |
| 4 | `test_one_strike_scores_bonus` | 한 프레임에서 스트라이크, 다음 두 롤 3·4, 나머지 거터 → `score() == 10 + 3 + 4 + 3 + 4 = 24` | 스트라이크 보너스(다음 2롤 가산) 로직은 아직 검증되지 않음 |
| 5 | `test_perfect_game_scores_300` | 12회 연속 스트라이크 → `score() == 300` | 10프레임 경계 + 필 볼(11·12번째 롤)이 마지막 프레임 보너스로만 쓰이고 별도 프레임을 만들지 않는지 검증 |

5개 테스트를 모두 통과시키면 kata의 스코어링 규칙(합산, 스페어, 스트라이크, 10프레임 필 볼)이 전부 커버된다.

## Human-in-the-Loop 체크포인트

1. **PLAN 승인** — 이 문서를 사람이 검토 후 승인해야 RED 1 진행.
2. **RED 승인** — 각 테스트를 Plan대로 작성했는지 확인 후 GREEN 진행.
3. **GREEN 승인** — 최소 구현인지(과도한 일반화 없는지) 확인 후 다음 RED 진행.
4. **REFACTOR 승인** — 동작 변경 없이 정리됐는지 확인.

각 RED 단계 커밋에는 그 시점의 `Plan.md`를 함께 커밋하고, 각 GREEN 리뷰 후에는 구현 코드를 커밋한다.
